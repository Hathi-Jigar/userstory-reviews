# User Story Reviews

Review space for digiQC user stories. Reviewers comment inline before stories are moved to the product-management repo.

## How It Works

1. Reviewer opens the review link
2. First visit: enters name + access key (saved in browser, one-time)
3. Selects text in the spec → comment popup appears → submits comment
4. Comments are stored in `comments.json` in this repo
5. Author sees comments, makes changes, resolves comments
6. Reviewer clicks **Approve** or **Changes Needed**
7. Approved stories move to the `product-management` repo
